01OOP Features & OOP vs POP Every Year ▶
Object-Oriented Programming (OOP) is a programming paradigm that organizes software design around data (objects) rather than functions and logic.

4 Pillars of OOP

  • Encapsulation – Wrapping data and functions together inside a class; hiding internal details.
  • Abstraction – Showing only necessary features and hiding implementation details.
  • Inheritance – A class (derived) acquiring properties of another class (base).
  • Polymorphism – One function/operator behaving differently in different situations.

Other Features

  • Classes and Objects – Class is a blueprint; Object is an instance.
  • Message Passing – Objects communicate by calling each other's member functions.
  • Dynamic Binding – Function call resolved at runtime using virtual functions.

OOP vs POP (Procedure-Oriented Programming)

02Reference Variables + Swap Program Every Year ▶
A reference variable is an alias (alternate name) for an already existing variable. It refers to the same memory location as the original variable.

Declaration Syntax

int a = 10;
int &ref = a;  // ref is a reference to a
ref = 20;      // now a = 20 also

Properties of Reference Variables

  • Must be initialized at the time of declaration.
  • Cannot be re-assigned to refer to another variable.
  • No separate memory is allocated – same as original variable.
  • Used mainly in function parameters (call by reference).

Uses of Reference Variables

  • Passing large objects to functions without copying (efficient).
  • Returning multiple values from a function.
  • Operator overloading.

Swap Program Using Reference Variables

#include<iostream>
using namespace std;

void swap(int &x, int &y) {
    int temp = x;
    x = y;
    y = temp;
}

int main() {
    int a = 10, b = 20;
    cout << "Before: a=" << a << ", b=" << b << endl;
    swap(a, b);
    cout << "After: a=" << a << ", b=" << b << endl;
    return 0;
}
// Output: Before: a=10, b=20
//         After: a=20, b=10
Dry run: before swap a=10,b=20 → after swap a=20,b=10. Write this in exam.
03Inline Functions Every Year ▶
An inline function is a function that is expanded in line (at the call site) by the compiler at compile time, avoiding function call overhead.

Syntax

inline int square(int x) {
    return x * x;
}

int main() {
    cout << square(5);  // compiler replaces this with: 5*5
    return 0;
}

Advantages

  • Eliminates function call overhead (no stack push/pop).
  • Faster execution for small, frequently called functions.
  • Works like a macro but with type checking.

Disadvantages

  • Increases code size (code bloat) if function is large.
  • Not suitable for functions with loops, recursion, or switch.
  • Compiler may ignore inline request for complex functions.

Inline Function vs Preprocessor Macro

Inline FunctionMacro (#define)
Processed by compilerProcessed by preprocessor
Type checking doneNo type checking
Safer and more reliableProne to errors (side effects)
Defined using inline keywordDefined using #define
Can be debuggedCannot be debugged easily

04Access Specifiers: private, public, protected High Chance ▶
Access specifiers in C++ control the visibility and accessibility of class members (data and functions).

Types

  • private – Members accessible only within the same class. Default for class.
  • public – Members accessible from anywhere (inside class, outside class, derived class).
  • protected – Members accessible within the same class AND derived classes, but NOT outside.

Comparison Table

Location of Accessprivateprotectedpublic
Same class✔ Yes✔ Yes✔ Yes
Derived class✘ No✔ Yes✔ Yes
Outside class✘ No✘ No✔ Yes

Example Program

#include<iostream>
using namespace std;

class Animal {
private:
    int age = 5;        // only inside Animal
protected:
    string name = "Dog"; // Animal + derived classes
public:
    void show() {
        cout << "Name: " << name << ", Age: " << age;
    }
};

class Dog : public Animal {
public:
    void display() {
        cout << name;  // OK: protected accessible in derived
        // cout << age; // ERROR: private not accessible
    }
};

int main() {
    Animal a;
    a.show();     // OK: public
    // a.age;    // ERROR: private
    // a.name;   // ERROR: protected
    return 0;
}

Why use protected?

When you want a member to be hidden from outside (like private) but still accessible to child classes (derived classes), you use protected. This is very useful in inheritance.

Draw the table in exam — it gets full marks in 2 minutes.
05Encapsulation & Data Hiding Every Year ▶
Encapsulation is the mechanism of binding data (variables) and the functions that operate on that data together into a single unit called a class.

Data Hiding means keeping the internal data private so it cannot be accessed directly from outside the class. Access is provided only through public member functions (getters/setters).

Example

#include<iostream>
using namespace std;

class BankAccount {
private:
    double balance;        // data hidden from outside
public:
    void setBalance(double b) { balance = b; }
    double getBalance()      { return balance; }
    void deposit(double amt)  { balance += amt; }
};

int main() {
    BankAccount acc;
    acc.setBalance(1000);
    acc.deposit(500);
    cout << "Balance: " << acc.getBalance();
    return 0;
}

Advantages

  • Protects data from accidental modification.
  • Increases code maintainability and flexibility.
  • Provides controlled access through getter/setter methods.
06Constructors & Copy Constructor Every Year ▶
A Constructor is a special member function that is automatically called when an object is created. It has the same name as the class and no return type.

Characteristics of Constructor

  • Same name as class.
  • No return type (not even void).
  • Called automatically when object is created.
  • Can be overloaded.

Types of Constructors

  • Default Constructor – No parameters.
  • Parameterized Constructor – Takes arguments to initialize members.
  • Copy Constructor – Creates a new object as a copy of an existing object.

Example – All Three Types

#include<iostream>
using namespace std;

class Distance {
private:
    int feet, inches;
public:
    // Default Constructor
    Distance() { feet = 0; inches = 0; }

    // Parameterized Constructor
    Distance(int f, int i) { feet = f; inches = i; }

    // Copy Constructor
    Distance(Distance &d) { feet = d.feet; inches = d.inches; }

    void show() {
        cout << feet << "ft " << inches << "in" << endl;
    }
};

int main() {
    Distance d1;           // Default
    Distance d2(5, 8);     // Parameterized
    Distance d3(d2);       // Copy
    d1.show(); d2.show(); d3.show();
    return 0;
}

Do we need Default Constructor when Parameterized exists?

YES — when you define a parameterized constructor, the compiler does NOT provide a default constructor automatically. So if you want to create objects without arguments, you must explicitly define the default constructor.

This is a frequently asked sub-question. Always answer it clearly.

A Destructor is a special member function called automatically when an object goes out of scope or is explicitly deleted. It cleans up resources.

08Friend Functions Every Year (2024, 2025) ▶
A friend function is a function that is not a member of a class but has access to the private and protected members of that class. It is declared using the keyword friend inside the class.

Characteristics

  • Not a member of the class but declared inside it.
  • Can access private and protected members.
  • Called without using an object (no dot operator).
  • Cannot be called using object.function() syntax.

Friend Function in Two Classes

#include<iostream>
using namespace std;

class ClassB;  // forward declaration

class ClassA {
private:
    int a = 10;
public:
    friend void show(ClassA, ClassB);
};

class ClassB {
private:
    int b = 20;
public:
    friend void show(ClassA, ClassB);
};

// friend function – not a member of either class
void show(ClassA obj1, ClassB obj2) {
    cout << "A = " << obj1.a << endl;
    cout << "B = " << obj2.b << endl;
}

int main() {
    ClassA a;
    ClassB b;
    show(a, b);
    return 0;
}
Always include forward declaration (class ClassB;) at the top. Examiners check for this.
09Operator Overloading Every Year ▶
Operator overloading allows C++ operators to be redefined to work with user-defined data types (classes). The keyword used is operator.

Rules of Operator Overloading

  • Only existing operators can be overloaded; new ones cannot be created.
  • Arity (number of operands) of operator cannot be changed.
  • At least one operand must be a user-defined type.
  • Precedence and associativity of operators cannot be changed.
  • Overloaded using keyword operator.

Operators that CANNOT be Overloaded

:: (scope resolution)  |  . (dot)  |  .* (pointer-to-member)  |  ?: (ternary)  |  sizeof

Binary Operator Overloading (+)

#include<iostream>
using namespace std;

class Complex {
public:
    int real, imag;
    Complex(int r=0, int i=0) { real=r; imag=i; }

    // Overloading + operator
    Complex operator+(Complex c) {
        return Complex(real + c.real, imag + c.imag);
    }

    void show() {
        cout << real << " + " << imag << "i" << endl;
    }
};

int main() {
    Complex c1(3, 4), c2(1, 2);
    Complex c3 = c1 + c2;  // calls operator+
    c3.show();             // Output: 4 + 6i
    return 0;
}

Unary Operator Overloading (++)

Complex operator++() {
    real++;  imag++;
    return *this;
}
Always mention which operators cannot be overloaded — examiners frequently ask this.
10Inheritance – All Types Every Year ▶
Inheritance is the mechanism by which a new class (derived/child class) acquires properties and behaviors of an existing class (base/parent class).

Advantages

  • Code reusability – no need to rewrite existing code.
  • Reduces redundancy.
  • Supports polymorphism.
  • Easy to extend functionality.

Types of Inheritance

TypeDescriptionSyntax
SingleOne derived from one baseclass B : public A
MultipleOne derived from multiple basesclass C : public A, public B
MultilevelChain: A → B → Cclass C : public B
HierarchicalMultiple derived from one baseclass B : public A; class C : public A
HybridCombination of above typesCombination

public vs private Derived Class

Base memberpublic inheritanceprivate inheritance
publicpublic in derivedprivate in derived
protectedprotected in derivedprivate in derived
privatenot accessiblenot accessible

Example – Single Inheritance

class Animal {
public:
    void eat() { cout << "Eating" << endl; }
};

class Dog : public Animal {
public:
    void bark() { cout << "Barking" << endl; }
};

int main() {
    Dog d;
    d.eat();   // inherited from Animal
    d.bark();  // Dog's own function
}
11Polymorphism Every Year ▶
Polymorphism means "many forms." In C++, it allows functions or operators to behave differently based on the context (object or arguments).

Types of Polymorphism

  • Compile-time (Static) Polymorphism – Resolved at compile time.
    • Function Overloading
    • Operator Overloading
  • Runtime (Dynamic) Polymorphism – Resolved at runtime.
    • Virtual Functions (with inheritance)

Function Overloading Example

#include<iostream>
using namespace std;

class Area {
public:
    void calculate(float r) {
        cout << "Circle area: " << 3.14 * r * r << endl;
    }
    void calculate(float l, float b) {
        cout << "Rectangle area: " << l * b << endl;
    }
    void calculate(float b, float h, int) {
        cout << "Triangle area: " << 0.5 * b * h << endl;
    }
};
Polymorphism = compile-time + runtime. Always name both types with examples.
12Virtual Functions Every Year ▶
A virtual function is a member function in the base class that can be overridden in a derived class. It enables runtime polymorphism. Declared using the keyword virtual.

Why Virtual Functions are Essential

Without virtual functions, if you call a function using a base class pointer, the base class version is always called (static binding). With virtual, the correct derived class version is called at runtime (dynamic binding).

Complete Program

#include<iostream>
using namespace std;

class Shape {
public:
    virtual void draw() {    // virtual function
        cout << "Drawing Shape" << endl;
    }
};

class Circle : public Shape {
public:
    void draw() override {
        cout << "Drawing Circle" << endl;
    }
};

class Rectangle : public Shape {
public:
    void draw() override {
        cout << "Drawing Rectangle" << endl;
    }
};

int main() {
    Shape *ptr;            // base class pointer

    Circle c;
    ptr = &c;
    ptr->draw();           // Output: Drawing Circle (runtime!)

    Rectangle r;
    ptr = &r;
    ptr->draw();           // Output: Drawing Rectangle

    return 0;
}

Pure Virtual Function (Abstract Class)

virtual void draw() = 0;  // pure virtual – class becomes abstract

A class with at least one pure virtual function is called an abstract class. Objects of abstract class cannot be created.

Use a base class pointer in the demo — that's the whole point of virtual functions. Examiners check for this.
13Exception Handling Every Year ▶
Exception handling is a mechanism to handle runtime errors (exceptions) so that the program does not crash. C++ uses three keywords: try, catch, and throw.

Keywords

  • try – Block of code that might throw an exception.
  • throw – Used to throw an exception when an error occurs.
  • catch – Block that catches and handles the exception.

Advantages

  • Separates error-handling code from normal code.
  • Prevents program from crashing on runtime errors.
  • Multiple exceptions can be handled with multiple catch blocks.
  • Makes code more readable and maintainable.

Example Program

#include<iostream>
using namespace std;

int divide(int a, int b) {
    if (b == 0)
        throw "Division by zero!";  // throw exception
    return a / b;
}

int main() {
    try {
        cout << divide(10, 2) << endl;   // OK: outputs 5
        cout << divide(10, 0) << endl;   // throws exception
    }
    catch (const char *msg) {
        cout << "Error: " << msg << endl;
    }
    return 0;
}
// Output: 5
//         Error: Division by zero!

Multiple Catch Blocks

try { throw 10; }
catch(int e)    { cout << "Int exception: "    << e; }
catch(double e) { cout << "Double exception: " << e; }
catch(...)      { cout << "Unknown exception"; }
14Static Binding vs Dynamic Binding High Chance ▶
Binding refers to linking a function call to the function body. It can happen at compile-time (static) or runtime (dynamic).

Also calledEarly bindingLate binding
Used withNormal functions, overloadingVirtual functions

Late Binding Example

Shape *ptr = new Circle();
ptr->draw();  // resolved at runtime → calls Circle::draw()

This is late binding — the exact function to call is decided at runtime based on the actual object type pointed to by ptr.

15Structure vs Class in C++ High Chance ▶
Both struct and class are used to define user-defined data types. The main difference is the default access specifier.
FeatureStructure (struct)Class
Default accesspublicprivate

Example – Structure in C++

struct Student {
    int rollno;      // public by default
    string name;
    void show() { cout << rollno << " " << name; }
};

int main() {
    Student s;
    s.rollno = 1;
    s.name = "Ali";
    s.show();
}

Structures are called heterogeneous data types because they can group variables of different data types under one name.
